<h3 id="creating-image-profiles">Creating Image Profiles</h3>
<p>Image Profiles are used to specify software image details.  As part of POAP process, software images are downloaded by switch.  To download software image, following information is required:
    image: filename which identifies software image
    imageserver_ip: IP address of server where the image is located
    username: username required to login to this server
    password: password required to login to this server
  access_protocol: IP protocol (tftp, ftp, scp, http) needed to retrieve the image</p>
<p>Once a profile is defined, it can be applied to Spine tier or Leaf tier switches of a fabric.
Image profiles are stored in directory ignite/fabric/image_profile.py. Three profiles &quot;spine_image&quot;, &quot;leaf_image&quot;, and &quot;default_image&quot; are pre-defined in this file.  The default_image profile is used to bring up switches when user has not selected a specific image profile.  New image profiles can be added to this file.</p>
<p>To apply an image profile to a fabric tier, use &quot;Set Defaults&quot; in &quot;Fabrics&quot; view.
Selected image profile contents will be sent to the switch in response to the request which it sends to Ignite server for building and downloading startup configuration.  Ignite_poap.py the example poap.py script extracts the image profile and uses it download the software image.</p>
